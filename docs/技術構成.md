# 技術構成

## フロントエンドスタック

| 技術 | 選定理由 |
|------|---------|
| Vite | 高速なビルド・HMR、フロントエンド完結型に最適 |
| React | コンポーネント設計とステート管理 |
| TypeScript | 型安全性、アルゴリズム実装の信頼性確保 |
| Tailwind CSS | atcoder.jpのデザインを模倣した迅速なスタイリング |

## データ永続化

**ローカルストレージのみ**。サーバー不要・アカウント登録不要。

競プロユーザーはPC環境を前提としており、デバイス跨ぎの同期は要件外。

保存内容:
- 測定履歴（日付・適正レート・測定結果詳細）
- 出題済み問題リスト（再出題防止）
- セッション中断時のスナップショット（再開用）

## 外部API

**AtCoder Problems API**（kenkoooo製・非公式）

- 利用目的: 問題の難易度データ取得、ユーザーのAC済み問題取得
- 制約: リクエスト間隔1秒以上
- リスク: 非公式サービスのため、予告なしのAPI変更・廃止の可能性あり
- 正式な利用規約なし（非公式ガイドラインに準拠）
