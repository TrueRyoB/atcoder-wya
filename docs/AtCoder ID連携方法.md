## AtCoder ID連携方法

### 概要

ユーザが AtCoder ID を入力したとき、AtCoder Problems API を用いて最新レートとコンテスト参加回数を取得する。

### データソース

AtCoder Problems API（非公式・CORS許可済み）

```
GET https://kenkoooo.com/atcoder/atcoder-problems/api/v3/user/rating?user={atcoder_id}
```

レーティング履歴が配列形式で返される。各要素は1回の rated コンテスト結果に対応する。

### 取得する値

| 値 | 取得方法 | 空配列時 |
|---|---|---|
| 最新レート | 配列末尾要素の `NewRating` | 1200（仮） |
| コンテスト参加回数 | 配列の `length` | 0 |
| 暫定フラグ | 参加回数 < 14 | true |

### 存在しないIDの扱い

「IDが存在しない」と「コンテスト未参加」はいずれも空配列 `[]` が返るため区別しない。
アプリの仕様上、現時点での実力の指標として活用できるデータがあるかどうかのみが重要なため、両者とも最新レート `1200`（仮）として扱う。

### 暫定レートについて

コンテスト参加回数が 14 回未満の場合、レートは**暫定**とみなす。

- 参加回数 0（未参加 / 存在しないID）：レート `1200`、暫定フラグ `true`
- 参加回数 1〜13：直近の `NewRating` を使用、暫定フラグ `true`
- 参加回数 14 以上：直近の `NewRating` を使用、暫定フラグ `false`

### 補足

コンテスト参加回数が十分でないときに過小評価されてしまうことは、次のmdに記す探索幅を調整することで対策する。
